To capture the baseline
------------------------
**Option A — GitHub Actions (recommended):**
1. Open https://github.com/Hansiongs/hans-backtest/actions/workflows/mutation.yml
2. Click "Run workflow" ▾ (dropdown on the right)
3. Branch: main
4. Click "Run workflow"
5. After completion (~30-60 min), download the `mutmut-report`
   artifact from the workflow run
6. Extract `mutation_score.txt` — this is the baseline score
7. Update the "Expected score" section below with the captured value

**Option B — Local (Linux/WSL required; mutmut does not support Windows):**
    MUTMUT_RUNNING_IN_BATCH=1 mutmut run --max-children 4
    mutmut stats

Acceptance threshold
--------------------
The mutation score should not drop by more than 5 percentage points
between releases. If it does, the new code is insufficiently tested
and needs additional coverage before release.

CI enforcement (planned for v0.5.x)
------------------------------------
The workflow `.github/workflows/mutation.yml` will be enhanced to:
1. Parse the baseline score from this file
2. Compare current run's score against baseline
3. Fail the workflow if the drop exceeds the 5pp threshold

Until the baseline is captured, this enforcement is a no-op.
