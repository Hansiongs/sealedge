Status (Phase 4 -- v0.5.0)

Baseline NOT yet captured. The CI workflow runs weekly on Ubuntu and
produces the baseline automatically on first execution. This file
documents the capture procedure and acceptance threshold.

To capture the baseline
------------------------
**Option A -- GitHub Actions (recommended):**
1. Open https://github.com/Hansiongs/hans-backtest/actions/workflows/mutation.yml
2. Click "Run workflow" (dropdown on the right)
3. Branch: main
4. Click "Run workflow"
5. After completion (~30-60 min), download the `mutmut-report`
   artifact from the workflow run
6. Extract `mutation_score.txt` -- this is the baseline score
7. Update the `BASELINE_SCORE_PCT` field below with the captured value
8. Commit the updated mutation_baseline.txt -- CI will then enforce
   the 5pp threshold automatically

**Option B -- Local (Linux/WSL required; mutmut does not support Windows):**
    MUTMUT_RUNNING_IN_BATCH=1 mutmut run --max-children 4
    mutmut stats

Acceptance threshold
--------------------
The mutation score must not drop by more than 5 percentage points
between the BASELINE_SCORE_PCT and any subsequent run. If it does,
the new code is insufficiently tested and needs additional coverage
before release. Enforcement: ``mutation.yml`` "Enforce 5pp threshold"
step (added in Sprint 3 fix 3.7).
